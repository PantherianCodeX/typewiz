# Draft log template

Use this template both:

- inside each rewritten ADR/spec (as its “Draft log” section), and
- as a mirrored file under `docs/_internal/draft_logs/<doc-id>.md`.

## Format (recommended)

- Keep each entry short (3–8 bullets).
- Each entry must reference:
  - mapping updates (MAP-/P-/CF-/OVL-/SUP- identifiers),
  - and the “why” at decision-level (not implementation-level).

- **Preservation:** <P-IDs and CF-IDs affected; verbatim/adapt decision>
- **Overlay:** <OVL-IDs applied; plan anchor>
- **Mapping:** <MAP-IDs updated; source→destination evidence>
- **Supersedence:** [SUP-IDs if any]
- **Notes / risks:** <open risks or required follow-up; Q-IDs if blocked>
